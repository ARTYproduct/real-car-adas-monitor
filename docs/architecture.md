# Архитектура системы мониторинга ADAS

Данный проект представляет собой программный комплекс для мониторинга телеметрии автомобиля и состояния водителя в реальном времени. Система объединяет обработку данных OBD-II и видеоаналитику (DMS) в едином графическом интерфейсе.

## Основные компоненты системы

### 1. Модуль телеметрии (OBD Domain)
* **OBDParser**: Компонент, отвечающий за чтение, парсинг и валидацию данных из внешних источников (CSV-файлы).
* **DrivingStyleClassifier**: Интеллектуальный модуль на базе ONNX Runtime, классифицирующий стиль вождения (нормальный, агрессивный, экономичный) на основе текущих показателей.

### 2. Модуль мониторинга водителя (DMS Domain)
* **DMSMonitor**: Анализирует видеопоток с веб-камеры с использованием библиотек OpenCV.
* **Детекция состояний**: Определение усталости (частота моргания/закрытые глаза) и отвлечения внимания.

### 3. Графический интерфейс и вывод (UI Domain)
* **Dashboard**: Единое окно визуализации, отрисовывающее приборную панель (спидометр, тахометр) и статус водителя.
* **Visual Logger**: Система наложения текстовых и графических уведомлений поверх видеопотока.

## Потоки данных (Data Flow)
1.  **Поток данных машины**: Файл `.csv` -> `OBDParser` -> `Classifier` -> Глобальное состояние системы.
2.  **Поток видео**: Камера -> `OpenCV` -> `DMSMonitor` -> Рендеринг в `Dashboard`.
3.  **Синхронизация**: Многопоточная обработка данных для обеспечения плавности интерфейса.

## Стек технологий
* **Язык**: C++17
* **Сборка**: CMake
* **Библиотеки**: 
    * OpenCV (Визуализация и обработка изображений)
    * ONNX Runtime (Машинное обучение)
    * Google Test (Модульное тестирование)
